# Page Briefs

## Uso del documento

Este archivo convierte el sitemap en definiciones concretas de contenido.
Cada pagina debe tener un objetivo claro antes de disenar o programar.

## Plantilla sugerida por pagina

- Objetivo de la pagina
- Usuario principal
- Mensaje principal
- Secciones o bloques
- CTA principal
- CTA secundaria
- Recursos visuales necesarios
- Notas de contenido pendientes

## Paginas detectadas en el sitemap

### Home

- Objetivo: comunicar de forma inmediata que ofrece el sitio, generar una primera impresion solida y dirigir al usuario hacia la accion principal mas importante.
- Usuario principal: persona que llega por primera vez al sitio y quiere entender rapido que se ofrece, si le resulta relevante y cual es el siguiente paso.
- Mensaje principal: aqui encuentras una propuesta clara, visualmente cuidada y facil de explorar para conocer los servicios, descubrir contenido util y avanzar hacia contacto o compra.
- Secciones o bloques:
  - Hero principal con propuesta de valor, texto breve de apoyo y CTA visible
  - Bloque de introduccion corta sobre la marca, proyecto o enfoque
  - Resumen de servicios principales con 3 a 4 opciones destacadas
  - Bloque de beneficios o diferenciadores para generar confianza
  - Destacados del blog para mostrar contenido reciente o util
  - Vista previa de productos o acceso rapido a la tienda
  - Llamado final a contacto o accion principal
- CTA principal: ver servicios.
- CTA secundaria: contactar ahora.
- Recursos visuales necesarios:
  - mockup del hero de Home
  - imagen principal o composicion visual de portada
  - iconos o recursos graficos para servicios
  - tarjetas visuales para blog y tienda
  - referencias de botones y bloques destacados
- Notas de contenido pendientes: definir la propuesta de valor exacta en una frase y confirmar si la prioridad principal del Home sera captar contacto o dirigir a servicios.
