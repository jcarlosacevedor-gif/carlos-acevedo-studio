# Documentacion del proyecto

## Objetivo

Centralizar la organizacion del sitio antes de pasar a codigo.

## Estructura actual

- `sitemap/`
  - `sitemap.md`: mapa base del sitio y lista inicial de paginas.
- `content/`
  - `page-briefs.md`: objetivo, contenido y CTA de cada pagina.
- `decisions/`
  - `visual-direction.md`: criterios visuales para diseno en GIMP.
  - `project-decisions.md`: decisiones tomadas, pendientes y preguntas abiertas.

## Orden recomendado de trabajo

1. Ajustar el sitemap si cambia la estructura general del sitio.
2. Completar `content/page-briefs.md` para definir que lleva cada pagina.
3. Completar `decisions/visual-direction.md` mientras avanzas en GIMP.
4. Registrar decisiones y dudas en `decisions/project-decisions.md`.

## Regla de trabajo

No pasar a implementacion web hasta que exista claridad minima en:

- estructura de paginas
- objetivo por pagina
- bloques principales por pagina
- estilo visual general
- decisiones abiertas mas importantes
